Introducing Functions in Sour

Functions are reusable blocks of code that perform specific tasks in your Sour programs. They help you organize your code, improve readability, and avoid redundancy.

Defining Functions

Here's the general syntax for defining a function in Sour:

fun functionName(parameters) {
  // Function body (code to be executed)
  return value;  // Optional: return a value
}
  • **`fun`:** Keyword to declare a function.
  • **`functionName`:** A descriptive name for your function.
  • **`parameters`:** Optional comma-separated list of variables that the function can accept as input.
  • **`Function body`:** The code that gets executed when the function is called.
  • **`return`:** Optional statement that returns a value from the function.

Calling Functions

Once you define a function, you can call it by its name followed by parentheses:

// Example function to greet someone
fun greet(name) {
  print("Hello, " + name + "!");
}

// Calling the greet function
greet("Alice");  // Output: Hello, Alice!

In this example, the `greet` function takes a parameter named `name`, and when called with the value "Alice", it prints a personalized greeting.

Benefits of Functions (Optional)

  • **Code Reusability:** Functions allow you to write code once and use it multiple times throughout your program.
  • **Improved Readability:** Functions break down complex logic into smaller, manageable parts, making code easier to understand.
  • **Modular Design:** Functions promote modularity, allowing you to organize your code into well-defined units.
