Understanding Variables in Sour

Variables are like containers that hold data in your Sour programs. You can use them to store information and manipulate it throughout your code.

Declaring Variables

To create a variable in Sour, you use the `var` keyword followed by the variable name and an assignment operator (`=`) to assign a value:

var name = "Alice";  // String variable
var age = 30;          // Number variable
var isLoggedIn = true;  // Boolean variable

In these examples, `name`, `age`, and `isLoggedIn` are variable names, and the values assigned are "Alice", 30, and true, respectively.

Using Variables

Once you declare a variable, you can use its name to access the stored value in your code. Here's an example:

print("Hello, " + name + "!");

In this code, the `name` variable's value ("Alice") is used within the `print` statement to create a personalized greeting.

Naming Conventions (Optional)

While Sour is not strictly case-sensitive, it's recommended to follow good naming conventions for readability:

  • Use descriptive names that reflect the variable's purpose (e.g., `customerName`, `totalCost`).
  • Start variable names with lowercase letters.
  • Use camelCase for multi-word names (e.g., `numberOfItems`).

Variable Scope (Optional)

Sour may have different variable scoping rules depending on its implementation. Here's a general concept found in many programming languages (check your Sour documentation for specifics):

  • **Local Scope:** Variables declared within a function or block are only accessible within that scope.
  • **Global Scope:** Variables declared outside of functions or blocks are accessible throughout the program.

Understanding variable scope is crucial for avoiding naming conflicts and writing well-structured Sour code.
