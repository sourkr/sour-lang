Every variable and data structure in Sour Lang has a data type. Data types are important because they tell the compiler what you are allowed to do with that variable or data structure. In other words, what functions and properties it has.

In the last chapter, Sour Lang was able to tell in the previous example that customers has type: int. Sour Lang's ability to infer the data type is called type inference. customers is assigned an integer value. From this, Sour Lang infers that customers has numerical data type: int. As a result, the compiler knows that you can perform arithmetic operations with customers:

Example 
        var customers = 10
        
        // Some customers leave the queue
        customers = 8
        
        customers = customers + 3 // Example of addition: 11
        customers += 7 // Example of addition: 18
        customers -= 3 // Example of subtraction: 15
        customers *= 2 // Example of multiplication: 30
        customers /= 3 // Example of division: 10
        
        print(customers) // 10
+=, -=, *=, /=, and %= are augmented assignment operators. For more information, see Augmented assignments.

In total, Sour Lang has the following basic types:

Category Basic Types
Integers byte, short, int, long
Decimals float, double
Booleans bool
Char char
String str

With this knowledge, you can declare variables and initialize them later. Sour Lang can manage this as long as variables are initialized before the first read.

Basic Types 
        // Variable explicitly typed and initialized
        const e: str = "hello"
        
        // Variables can be read because they have been initialized
        print(e) // hello
