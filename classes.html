Introducing Classes in Sour

Classes are blueprints for creating objects in Sour. They define the properties (data) and methods (functions) that objects of that class will have.

Defining Classes

Here's the general syntax for defining a class in Sour:

class ClassName {
        // Properties (data associated with the class)
        var propertyName1;
        var propertyName2;

        // Methods (functions that operate on the class)
        fun methodName1(parameters) {
          // Method body
        }

        fun methodName2(parameters) {
          // Method body
        }
      }
  • **`class`:** Keyword to declare a class.
  • **`ClassName`:** A name for your class (descriptive and starts with a capital letter).
  • **`Properties`:** Variables that define the data associated with objects of this class.
  • **`Methods`:** Functions that define the behavior of objects of this class and can operate on the class properties.

Creating Objects

Once you define a class, you can create objects (instances) of that class using the `new` keyword:

// Example class for a person
class Person {
  var name;
  var age;

  fun greet() {
    print("Hello, my name is " + name + "!");
  }
}

// Creating an object of the Person class
var alice = new Person();
alice.name = "Alice";
alice.age = 30;

alice.greet();  // Output: Hello, my name is Alice!

In this example, the `Person` class defines properties for name and age, and a `greet` method. We create an object named `alice` of the `Person` class and then set its properties and call its `greet` method.

Benefits of Classes (Optional)

  • **Object-Oriented Programming:** Classes promote object-oriented programming, a powerful paradigm for structuring complex applications.
  • **Code Reusability:** Classes allow you to create reusable templates for objects with similar properties and behaviors.
  • **Encapsulation:** Classes encapsulate data (properties) and behavior (methods) within a single unit, promoting data protection and modularity.
