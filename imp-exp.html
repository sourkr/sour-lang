Sharing Code with Imports & Exports

As your Sour programs grow larger, you may want to organize your code into separate files for better maintainability. Imports and exports allow you to share code between these files.

Exporting Code

The `export` keyword is used to make variables, functions, or classes available for use in other Sour files. Here's an example:

export var message = "Hello from test.sour!";

export fun greet() {
  print(message);
}

export class User {
  // Class definition
}
  • **`export`:** Keyword to declare elements for export.
  • **Exported elements:** You can export variables, functions, and classes.

Importing Code

The `import` keyword allows you to use exported elements from other Sour files in your current file. Here's how to import:

import { message, greet, User } from "test";  // Import from "test.sour"

// Using imported elements
print(message);
greet();

var user1 = new User();  // Create an instance of the imported User class
  • **`import`:** Keyword to initiate the import.
  • **`from`:** Keyword to specify the source file.
  • **Exported elements:** You can import specific elements (using curly braces) or import all exports using `*` (not recommended for large files).
  • **Source file:** The path to the file containing the exported elements. In Sour, the `.sour` extension is usually not required.

Importing Specific vs. All Elements

It's generally recommended to import only the specific elements you need from a file to avoid naming conflicts and improve code clarity.

File Paths (Optional)

The way you specify the source file path in the `import` statement might vary depending on your Sour project structure. Here are two common scenarios:

  • **Importing from the same directory:** 
    import { a } from "test.first"; // Assuming "test.first.sour" exists in the same directory
  • **Importing from a subdirectory:** 
    import { a } from "utils/test"; // Assuming "test.sour" exists in a subdirectory called "utils"

Consult your Sour documentation for specific guidelines on file paths for imports.
